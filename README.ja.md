# goal-me

*[English README is here](README.md)*

曖昧な「○○を作りたい／調べたい」を、**機械的に検証可能な閉ループ**へ落とし込む Claude Code スキルです。これにより、組み込みの `/goal` コマンドが **ブレず・コストを膨らませずに** ゴールを回しきれるようになります。

`grill-me` のシンプルさと *ループエンジニアリング* の考え方が下敷きです。エージェントは `Goal → Action → Check → Fix → Repeat until done` を回し、`/goal` は **あなたが書いた条件が実際に true になるまで** ループし続けます。条件が曖昧だとループは脱線し（"open loop"）、トークンを焼き続けます。`goal-me` は先にあなたへインタビューして、**経路の狭い閉ループ**と、客観的に判定できる精密な停止条件を設計します。

---

## なぜ使うのか

`/goal` は強力ですが、渡す条件の質がすべてです。「コードをきれいにして」と渡せば脱線するか延々と回り続けます。「`pytest tests/auth` が全 pass、`ruff` がエラー0」と渡せば、いつ終わりかを正確に判定できます。

`goal-me` は、その種の条件をあなたの代わりに作るフロントエンドです。適切な質問をし、あいまい語を禁止し、すべての「完了」を *別の* モデルが検証できる形（Maker ≠ Checker）へ強制します。

**開発・リサーチ・文章・調査**に使え、**新規・既存どちらのプロジェクト**にも対応します。

---

## 必要なもの

- プラグイン対応（`/plugin`）の [Claude Code](https://docs.anthropic.com/ja/docs/claude-code)。
- 組み込みの `/goal` コマンド（`goal-me` が設計した後、ループを回すのに使う）。`/goal-me` 自体は設計とファイル書き出しのみで、ループ実行は別の任意ステップです。

## インストール

```
/plugin marketplace add ecoopnet/goal-me
/plugin install goal-me
```

入っているか確認:

```
/plugin            # インストール一覧に goal-me が出る
/help              # コマンド一覧に /goal-me が出る
```

### 手動インストール（代替）

マーケットを使わない場合は、スキルフォルダをスキルディレクトリへコピー:

```
git clone https://github.com/ecoopnet/goal-me
cp -r goal-me/plugins/goal-me/skills/goal-me ~/.claude/skills/
```

### 更新 / アンインストール

```
/plugin marketplace update ecoopnet/goal-me   # 最新を取得
/plugin uninstall goal-me                      # 削除
```

## クイックスタート

1. どんなに曖昧でも、手元の状態で設計セッションを始める:

   ```
   /goal-me ポートフォリオのリスクを計算する小さなWebツールを作りたい、スマホ対応で
   ```

2. **インタビューに答える。** 1問ずつ、各質問に推奨回答つきで尋ねられます。リポジトリやファイルを読んで自分で分かることは訊かずに調べます。質問は次の点に収束します:
   - 「完了」とは何か（**観測可能な事実**として）
   - それを**どう機械的に検証するか**（テスト / lint / typecheck / grep / 数値基準 / 出典数 / チェックリスト）
   - 何が**スコープ外**か（コストを抑える鍵）
   - コストガードレールとエスカレーション条件

3. **ファイルが書き出される**。`.claude/goals/<slug>/` 配下に、ゴールの規模に応じて:
   - 小さいゴール → `GOAL.md` 1枚
   - 大きいゴール → `GOAL.md` + `STEPS.md` + `PROGRESS.md`

4. **コピペ用のキックオフ・コマンド**が提示される。例:

   ```
   /goal .claude/goals/<slug>/GOAL.md を読み、その「完了条件」が全て真になるまで実装。<エスカレーション条件>なら停止して確認。
   ```

5. **実行する。** `/goal` は条件が満たされるまでループし、満たされたら自動で停止します。定期実行は `/loop`。

---

## 書き出されるもの

```
.claude/goals/<slug>/
├── GOAL.md       # 正本: 完了条件・スコープ・コンテキスト・コストガードレール
├── STEPS.md      # (大規模) 経路: 各ステップに評価ゲート
└── PROGRESS.md   # (大規模) 実行をまたぐメモリ
```

`GOAL.md` の **完了条件チェックリストが正本（source of truth）** です。`/goal` はこれらのファイルを自動では読みません。キックオフ・コマンドで指し示します。コマンドを短縮しても完了判定が成立するよう、条件は `GOAL.md` 内で自己完結させます。

## 設計方針

- **既定は閉ループ。** まず人間が経路を狭く設計する。無制限な探索はコストを膨らませる。
- **機械判定可能な停止条件のみ。** あいまい語（「きれいに」「改善」「confidence高」）は禁止し、具体化を強制。
- **Maker ≠ Checker。** 停止条件は実行者とは別のモデルが観測可能な状態から判定する。だから客観的に判定可能でなければならない（自己採点にしない）。
- **成果物は規模に合わせる。** 1セッションで終わる作業に3ファイルは作らない。

## 構成

```
goal-me/
├── .claude-plugin/marketplace.json     # マーケットプレイス定義
└── plugins/goal-me/
    ├── .claude-plugin/plugin.json      # プラグイン定義
    └── skills/goal-me/
        ├── SKILL.md                    # 振る舞い（英語。インタビューと生成物はユーザーの言語に従う）
        └── reference/output-format.md  # ファイル雛形＋良い/悪いゴール例
```
